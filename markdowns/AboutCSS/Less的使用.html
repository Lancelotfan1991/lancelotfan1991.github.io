        # Less的使用

        案例取自于Less中文网。

        # 1，Less是什么？

        Less 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展。

        # 2，Less提供的扩展

        • 变量：variable

        变量的存在允许我们设置全局通用的样式或者字体颜色等等。

        @width: 10px;
@height: @width + 10px;

#header {
  width: @width;
  height: @height;
}
/**最终可以被编译为*/
#header {
  width: 10px;
  height: 20px;
}
        • 混合：mixins

        混合允许我们最大程度的复用css属性。避免写很多重复的代码。

        .bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
/*可以这样使用 */
#menu a {
  color: #111;
  .bordered();
}

.post a {
  color: red;
  .bordered();
}
        • 嵌套：nests

        嵌套允许我们重新组织css的结构，使其更加简介，也更容易理解。

        #header {
  color: black;
}
#header .navigation {
  font-size: 12px;
}
#header .logo {
  width: 300px;
}
/* 用less重写，可以将原本属于header的navigation和logo直接写在header中 */
#header {
  color: black;
  .navigation {
    font-size: 12px;
  }
  .logo {
    width: 300px;
  }
}
        • 运算:operations

        运算符允许我们对css进行一些数学运算方面的工作。

        // 所有操作数被转换成相同的单位
@conversion-1: 5cm + 10mm; // 结果是 6cm
@conversion-2: 2 - 3cm - 5mm; // 结果是 -1.5cm

// conversion is impossible
@incompatible-units: 2 + 5px - 3cm; // 结果是 4px

// example with variables
@base: 5%;
@filler: @base * 2; // 结果是 10%
@other: @base + @filler; // 结果是 15%

        # 3，在webpack中使用Less-loader

        在webpack的脚手架中使用less的话，需要设置LessLoader：

        这里需要注意三个loader的顺序：先用less-loader将Less转化为CSS，再用css-loader将css转化为commonjs，最后将js中的样式创建出来。

        //安装
npm install --save-dev less-loader less
// webpack.config.js
module.exports = {
    ...
    module: {
        rules: [{
            test: /\.less$/,
            use: [{
                loader: "style-loader" // creates style nodes from JS strings
            }, {
                loader: "css-loader" // translates CSS into CommonJS
            }, {
                loader: "less-loader" // compiles Less to CSS
            }]
        }]
    }
};
