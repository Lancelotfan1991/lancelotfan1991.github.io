    # Redux的拆分

    # 1，上一篇中的一些问题：

    在Redux-1-原理中，我们使用了一个文件阐述Redux的基本使用，包栝：

    ------>定义初始状态

    ------>创建reducer

    ------>通过redux.createStore创建store

    ------>创建actions

    ------>通过state.subscribe订阅state的状态变化

    ------>通过store.dispatch派发相应的action

    但是这种做法存在一定的不足之处：

    第一：actions是写死的，每修改一个state，就必须创建一个action

    第二：不同模块耦合，不宜与后期维护和阅读。

    第三：通过字符串匹配action，容易匹配错误。

    因此，我们对上一篇的代码进行了优化

    # 2，拆分后的代码：

    # 1）reducer

    const initialState = {
	counter:0
}

export function reducer(state = initialState,action){
    switch(){
        case ADD_NUMBER:
        return {...state,counter:state.counter+action.num};
        case SUB_NUMBER:
        return {...state,counter:state.counter-action.num};
        default:
        return state;
    }
}

    # 2)constants.js

    export const ADD_NUMBER = 'ADD_NUMBER';
export const SUB_NUMBER = 'SUB_NUMBER';

    # 3)actionCreator

    import {ADD_NUMBER , SUB_NUMBER } from 'constants.js'
//将action写成函数
export function addAction = (num)=>({type : ADD_NUMBER,num});
export function subAction = (num)=>({type : SUB_NUMBER,num});

    # 4)index.js

    import reducer from './reducer'
import redux from 'redux';

const store = redux.createStore(reducer);

    上面这四个文件，存放在store文件夹下。

    当我们需要使用状态时，可以这样使用：

    import store from 'store/index.js'
import { addAction , subAction } from './store/actionCreator.js'

//订阅
store.subscribe(()=>{
    console.log(store.getState());
})
//派发事件
store.dispatch(addAction(1));	// { counter : 1}
store.dispatch(subAction(2));	// { counter : -1}
