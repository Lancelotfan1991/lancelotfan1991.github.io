# ESlint-ts和Webpack处理Typescript

# 1，babel事实上可以直接处理typescript

{
	test: /\.ts?/,
	exclude: /node_modules/
	use: 'babel-loader'
}

前提：

安装preset-typescript，并且在babel.config.js中进行配置

//安装：
npm intstall @babel/preset-typescript

//配置：
{
	presets: [
		["@babel/preset-env", {
			useBuiltIns: 'usage'
		}],
		["@babel/preset-react"],
		["@babel/preset-typescript"]
	]
}

# 2，ts-loader和babel-loader的选择：

ts-loader会对代码进行严格的检测，遇到类型错误等问题会帮助我们。

而babel-loader依旧可以编译通过。

而babel-loader的优势则在于可以转化polyfill。

事实上，typescript官方文档说明了：

可以利用babel-loader进行编译，而使用tsc进行校验。

//使用如下指令可以在进行typescript校验时，不输出多余的js文件。
tsc --noEmit

// 如果希望实时地校验typescript
tsc --noEmit --watch

# 3，什么是ESLint

是一个静态代码分析工具。

# 4，如何使用ESLint

npx eslint  --init

配置完毕后，会生成.eslintrc.js

# 5，一些常见的eslint约束项。

no-unused-vars: 'off'
eof-last: ''

# 6，在Webpack配置文件中使用eslint

rules: [
	{
		test: /\.jsx?$/,
		exclude: /node_modules/,
		use: [
			"babel-loader",
			"eslint-loader"
		]
	}
]
