    # 2，babel是什么？

    babel本质上是个工具链。主要用于旧浏览器或者环境中，将ECMAScript 2015+代码转换为向后兼容版本的Javascript。

    包栝： 语法转换，源代码转换，Polyfill实现目标环境缺少的功能等。

    # 3，如何使用。

    npm install @babel/core -d

    想要直接通过命令行工具，使用babel，还需要：

    npm install @babel/cli -d

    之后可以通过babel直接处理对应的js文件：

    npx babel src --out-dir result

    # 4，一些常用的babel插件

    # （1）箭头函数转化插件：plugin-transform-arrow-function

    可以将箭头函数转化为function。

    # （2）const转化为var：plugin-transform-block-scoping。

    可以将const转化为var。

    # 5，直接使用预设。

    npm install @babel/preset-env -d

    这个命令相当于一系列插件的合集。

    安装预设后，再使用babel：

    npx babel src --out-dir result --preset=@babel/preset-env

    # 6，babel的底层原理

    我们平常编写的代码将会首先parsing 转化为ast（Abstract syntax tree，抽象语法树），再transfomation转化为字节码（Byte Code），最后由v8引擎处理。

    因此babel事实上可以看做成一种javascript编译器。

    # 7，在Webpack中使用babel

    （1）安装babel-loader

    npm install babel-loader -d

    （2）在webpack-config.js中设置对应的rules

    module: [
	rules: [
		{
			test: /\.js$/,
			use: {
				loader: 'babel-loader',
				options: {
					plugins: [
						"@babel/plugin-transform-arrow-function",
						"@babel/plugin-transfrom-block-scoping"
					]
					
					// 或者直接使用预设
					preset： [
						"@babel/preset-env"
					]
				}	
			}
		}
	]
]

    （1）通常意义上的broswerlistrc文件

    （3）我们还可以在webpack.config.js中，直接设置目标：

    这时，webpack.config.js中的配置是优先生效的。

    presets: [
	["babel/preset-env", {
		targets: ["chrome 88"]
	}]
]

    # 9，polyfill

    polyfill类似于一种补丁，可以帮助我们更好地使用javascript。

    比如：Promise，Generator，Symbol以及某些实例方法比如Array.prototype.includes等特性。

    如何设置webpack如何使用polyfill？

    可以通过useBuiltIns来控制：

    presets: [
	["babel/preset-env", {
		// useBuiltIns: 可以设置下面三个值：
		// false: 不用任何polyfill相关的代码
		// usage：代码中需要哪些polyfill，就使用哪些相关的api
		// entry：需要自己手动在入口文件中添加corejs和regenerator-runtime,相应的代码体积也会加大。这个时候，会跟据目标浏览器引用。
		useBuiltIns: ["chrome 88"]，
		corejs: 3
	}]
]

    当设置为entry时，需要添加的代码：

    import  'core-js/stable';
import	'regenerator-runtime/runtime';

    # 10，plugin-transform-runtime

    当我们编写第三方库或者写工具时，不应当希望使用的polyfill污染全局。所以建议使用plugin-transform-runtime。

    # 12，ts-loader

    typescript已经提供了typescript compiler，用于直接转化typescript。

    tsc index.ts

    通过以下指令可以生成tsconfig.json：

    tsc --init

    然后在webpack.config.js中使用：

    {
	test: /\.ts$\/,
	use: "ts-loader"
}
